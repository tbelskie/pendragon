# Roadmap

## Phase 0: Notebook

Status: current.

- Create Markdown warroom.
- Create product rooms for the four portfolio products.
- Use GitHub Pages for easy viewing.
- Use templates for launch, docs, pricing, and retrospectives.
- Run the first real product audits.

## Phase 1: Internal Workflow

- Apply Pendragon to Dot Primer, Writing Quest, Vinegar, and MockBizOps.
- Document every decision and launch artifact.
- Identify repeated workflows.
- Turn repeated workflows into reusable playbooks.
- Publish selected build notes publicly if useful.

## Phase 2: Lightweight App

- Products dashboard.
- Launch rooms.
- Decision ledger.
- Docs checklist.
- GTM experiment tracker.
- Weekly review.
- Markdown export.
- GitHub issue export.

## Phase 3: Repo-Aware Warroom

- Connect GitHub repos.
- Pull README, package metadata, release state, docs surfaces, and issue status.
- Generate launch readiness reports.
- Generate docs plans.
- Create launch tasks.

## Phase 4: Codex Plugin

- Pull Pendragon product context into Codex.
- Ask Codex to audit repos against launch rooms.
- Send findings, tasks, docs drafts, and PR summaries back to Pendragon.
- Turn Pendragon into the strategic memory above repo-native work.

## Phase 5: Founder Product

- Open Pendragon to other micro-SaaS founders.
- Position around launch readiness, docs discipline, and GTM execution.
- Keep the promise narrow: turn half-built projects into launched products.
