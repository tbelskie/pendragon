# v0.14 Product Setup And Stage Engine Plan

Pendragon needs a generic platform spine before it can dogfood every portfolio product. Writing Quest, Dot Primer, Vinegar, MockBizOps, and Pendragon itself should not require one-off rooms.

The next product move is:

> Capture the product's reality, infer the right operating mode, and route the room accordingly.

## Goal

Ship a local-only Setup tab that turns founder answers into a stage diagnosis.

The stage engine should determine:

- current product stage
- product type
- business model
- current goal
- traction or proof signal
- source-of-truth link
- missing launch assets
- recommended room modules
- recommended next action stack
- default Forge surface type

## Stage Coverage

v0.14 expands the stage model beyond the first five modes:

- Idea
- Prototype
- Beta
- Launch
- Shipped
- Stalled
- Relaunch
- Growth

This matters because a founder with a dormant Chrome extension is not in the same state as a founder with an unfinished desktop beta. The platform has to route both without pretending everything is "paid beta."

## Product Setup Fields

The first setup surface captures:

- product name
- product type
- business model
- current goal
- stage override or inferred stage
- target date
- status
- source link
- working prototype signal
- repo/source signal
- user-access signal
- charging signal
- traction or proof note
- current focus

## Stage Engine Rules

v0.14 keeps inference intentionally simple:

- explicit stage override wins
- relaunch goal plus traction points to Relaunch
- relaunch goal without traction points to Stalled
- grow goal points to Growth
- monetization goal with user access points to Launch
- beta goal with prototype points to Beta
- prototype goal with prototype points to Prototype
- validation without prototype points to Idea

The point is not perfect classification. The point is to make product stage discussable and editable.

## Room Output

The setup diagnosis outputs:

- recommended mode
- recommended launch surface
- current goal
- setup completeness
- missing assets
- room module mix
- readiness bias
- recommended action stack

## Why This Matters

Pendragon stops being a prettier launch notebook when the product room can adapt to the founder's actual stage.

The wedge remains launch surface generation, but the moat starts here:

> Pendragon knows what kind of product situation it is operating in before it generates advice or assets.

## Acceptance Criteria

- A Setup tab exists in the app shell.
- Setup saves locally for the active product.
- Setup can infer stage from product signals.
- Saved setup updates product stage, status, target date, source link, and current focus.
- Warroom shows a Stage Engine panel.
- Forge uses expanded stage templates for idea, prototype, beta, launch, relaunch, stalled, shipped, and growth products.
- Seeded portfolio products include initial setup profiles.
