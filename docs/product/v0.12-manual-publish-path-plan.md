# v0.12 Manual Publish Path Plan

Forge can generate a launch page, score it, and produce a share package. The next step is to show the founder how to get the generated page onto the internet without building a deployment platform yet.

The principle:

> Make the manual path obvious before automating it.

## Product Bet

Solo founders do not need OAuth, hosting integrations, and deployment configuration on day one. They need a clear bridge from generated HTML to a real static page.

Manual Publish Path gives the founder:

- suggested static folder
- expected `index.html` location
- public URL pattern
- step-by-step publish checklist
- QA blockers carried into the publish path
- Markdown publish guide export

This makes Forge more useful immediately while avoiding connector drag.

## v0.12 Scope

Add Manual Publish Path to Forge:

- derive a static folder from the product name
- show the expected file location
- show a placeholder GitHub Pages URL pattern
- list manual publish steps
- flag CTA and QA blockers before broad traffic
- export a Markdown publish guide
- include publish steps in the existing share package export

## Non-Goals

Do not add:

- OAuth
- GitHub API publishing
- Netlify/Vercel integration
- file upload
- domain configuration
- analytics
- automatic commits

Those belong after the manual workflow proves useful.

## Writing Quest Dogfood Standard

Writing Quest should get a publish path that says:

- put the generated page at `launch/writing-quest/index.html`
- do not send broad traffic while the CTA URL is missing
- do not send broad traffic while critical docs and readiness gate remain blocked
- use the page privately as a staging draft until then

This lets us dogfood the launch surface without overpromising that it is ready for strangers.

## Success Criteria

- Forge shows Manual Publish Path after Launch Share Package.
- The section includes static folder, page file, and URL pattern.
- The path includes six practical steps.
- Markdown publish guide exports successfully.
- Share package Markdown includes the manual publish path.
- The feature remains local-only and connector-free.
