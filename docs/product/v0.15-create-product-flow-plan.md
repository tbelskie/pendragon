# v0.15 Create Product Flow Plan

v0.14 made Setup the stage router for existing products. v0.15 turns that router into a real product creation flow.

The next product move is:

> A founder can create a new product room from setup signals and immediately get a usable warroom.

## Goal

Ship a local-only "New product" flow that creates a complete starter product room.

The room should include:

- inferred stage
- setup profile
- one-liner and audience
- brief starter fields
- first decision
- starter docs assets
- source evidence link when supplied
- readiness rows
- next actions
- Forge surface type based on stage

## Entry Point

The sidebar portfolio gets a persistent New product control.

Clicking it opens Setup in create mode.

This avoids adding a broad settings area or full onboarding wizard. Product creation belongs beside the stage engine because the same answers power both.

## Required Field

Only product name is required.

Everything else can be incomplete because Pendragon should be useful in messy founder reality. Missing fields become surfaced setup gaps rather than blockers to creating the room.

## Generated Starter Room

When a room is created, Pendragon generates:

- a unique product id from the product name
- stage from the setup profile
- default status from the stage definition
- starter readiness rows
- starter docs assets appropriate to the stage
- one source evidence item if a source link is provided
- one first decision
- top three actions from the stage engine

## Non-Goals

Do not add:

- accounts
- cloud sync
- product deletion
- product duplication
- templates marketplace
- full onboarding wizard

Those can come later. v0.15 is about proving that the platform can create a room, not just edit seed data.

## Acceptance Criteria

- Sidebar has a New product control.
- New product opens a create-mode Setup screen.
- Product name is required.
- Source link validates full `http` or `https` links.
- Creating a product appends it to local workspace products.
- The new product becomes the active room.
- The generated room has setup profile, docs assets, evidence, decision, readiness, and next actions.
- The created product persists after reload through local storage.
- Forge renders for the created product.
