# v0.18 Clear First State Plan

v0.17 introduced a generated Clear First queue. v0.18 gives the founder light control over that queue without turning Pendragon into a generic task manager.

The next product move is:

> A founder can annotate and resolve command items while Pendragon keeps generating them from product truth.

## Goal

Add local state to Clear First items:

- founder note
- open or resolved status
- resolved timestamp
- reopen behavior

This lets the founder quiet handled work and preserve context while the underlying product-room signals are still being cleaned up.

## Product Boundary

Resolved does not mean the underlying product issue disappeared.

It means:

> The founder has handled this command item locally for now.

If the source condition still exists, Pendragon can regenerate the item if it is reopened or if the founder resets local state. The product room remains the source of truth.

## State Model

Each generated Clear First item receives a stable key from its source:

- `setup-stage-mismatch`
- `gate-blocker-*`
- `docs-critical-*`
- `forge-qa-*`
- `evidence-proof-gap`
- `decision-*`
- `setup-gap-*`
- `gate-risk-*`
- `warroom-momentum`

Product rooms store lightweight `clearFirstStates` entries keyed by that generated key.

## UX

Open queue items support:

- adding a founder note
- saving the note
- marking the item resolved
- opening the source module

Resolved items move out of the active queue and into a Recently resolved section with:

- note editing
- reopen
- source module routing

## Non-Goals

Do not add:

- due dates
- assignees
- drag-and-drop task ordering
- notification rules
- external issue sync
- bulk actions

Those would make sense only after Pendragon proves the generated command queue is useful during real dogfooding.

## Acceptance Criteria

- Clear First items have stable keys.
- Product rooms persist `clearFirstStates` in local storage.
- Founder notes can be saved per item.
- Mark resolved removes an item from the open queue.
- Recently resolved items can be reopened.
- Resolved items still preserve notes.
- Product cards show the top open Clear First item, not a resolved one.
- App assets bump to `0.18.0`.
