# v0.16 Portfolio Command Center Plan

Pendragon has enough room-level modules to become useful, but the founder still needs one place that answers the portfolio question:

> Which product needs my attention next?

v0.16 adds the first Portfolio Command Center.

## Goal

Create a portfolio-level view that ranks product rooms by attention need and routes the founder into the right next room.

This is the first step toward Pendragon becoming an operating system for a small product portfolio rather than a set of isolated launch trackers.

## Attention Model

Each product gets an attention score from:

- inverse launch readiness
- readiness gate level
- critical docs gaps
- unresolved decisions
- setup/stage engine gaps
- missing proof links
- inferred stage mismatch

The score is intentionally directional. It should help a founder choose the next focus, not pretend to be a precise business intelligence metric.

## Portfolio Summary

The command center shows:

- total product rooms
- average launch readiness
- blocked readiness gates
- stage mismatches
- products with proof attached

These are the first portfolio health metrics. They make Pendragon useful even before cloud sync, connectors, or team workflows exist.

## Routing

Each product card gives direct actions:

- open Warroom for current focus and next actions
- open Setup when signals or stage are wrong
- open Forge when the launch surface is the next asset to build

The founder should not have to remember which module contains the next move. The portfolio view should route them.

## Non-Goals

Do not add:

- cloud accounts
- team assignment
- external integrations
- charts that require analytics collection
- editable portfolio scoring weights
- automatic product deletion or archiving

v0.16 is about attention routing. Automation can come later.

## Acceptance Criteria

- Navigation includes Portfolio.
- App opens to Portfolio by default.
- Portfolio renders a command hero with a recommended focus product.
- Portfolio shows summary metrics across all product rooms.
- Product cards are sorted by attention score.
- Product cards show readiness, gate score, docs status, and proof count.
- Product cards explain why the product is being surfaced.
- Product card actions open Warroom, Setup, and Forge for that product.
- Header switches to portfolio mode with average readiness.
- Existing product-specific rooms continue to work.
- GitHub Pages loads the new app code without stale asset cache.
