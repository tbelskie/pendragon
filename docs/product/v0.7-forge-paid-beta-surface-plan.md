# v0.7 Forge Paid Beta Surface Plan

v0.6 proved the core Forge loop:

> Product room in, launch surface out.

v0.7 makes the output more commercially useful by giving Forge a stage-aware surface type. For Writing Quest, that surface is a paid beta page.

## Product Bet

The fastest way to make Pendragon real is to help a solo founder produce the page that asks for money.

The next Forge job is not a prettier preview. It is:

> Turn launch-room truth into a paid beta page draft that exposes the exact trust gaps blocking revenue.

## v0.7 Scope

Ship stage-aware Forge templates:

- `beta`: paid beta page
- `growth`: relaunch page
- `prototype`: validation page

Each template should define:

- surface type
- conversion goal
- primary CTA
- offer label
- proof/trust frame
- hard-truth risk
- questions the founder must answer before widening the launch

## Writing Quest Dogfood Standard

For Writing Quest, Forge should answer:

> Could this page credibly ask a writer for paid beta money today?

If the answer is no, Forge should explain why:

- save/export trust is blocked
- pricing is undecided
- install/support path needs proof
- launch docs are still open

That is the moat direction: Pendragon does not merely generate copy. It turns launch readiness into an external surface and shows what would break user trust.

## Generated Paid Beta Page Sections

The v0.7 paid beta surface includes:

- hero and gate verdict
- audience, surface type, and target date
- beta offer
- primary CTA
- problem, promise, milestone, pricing, and constraint
- beta trust assets
- launch gaps
- buyer questions
- next actions
- support readiness

## Non-Goals

Do not add:

- live payments
- hosted publishing
- editable template designer
- AI rewrite flow
- authentication
- custom domains

Those are downstream. The immediate goal is to make the internal page draft useful enough that Writing Quest can use it as the skeleton for a real paid beta surface.

## Success Criteria

- Writing Quest renders as a `Paid beta page`.
- The primary CTA is `Join paid beta`.
- The generated surface shows buyer questions before the founder asks for money.
- Trust-sensitive docs assets appear separately from generic proof.
- HTML export includes the same paid beta strategy and questions.
- Prototype and growth products still get appropriate default surface types.

## Next Step

After v0.7, Pendragon should dogfood the output against Writing Quest:

- add proof links for Writing Quest trust docs
- complete the save/export trust page
- choose beta pricing
- export the HTML draft
- compare it to the real Writing Quest marketing page we want to publish
