# Pendragon v0.3 Launch Room Plan

v0.3 turns Pendragon from a single warroom screen into the first real launch room.

The product remains local-first, backend-free, and dogfood-driven. The goal is to make the Writing Quest launch room useful enough for weekly product work while keeping the surface premium and calm.

## Product Job

Pendragon v0.3 should answer:

> What are we launching, what have we decided, what knowledge surface do we need, and what is ready to turn into a launch site?

## Required Scope

### Launch Room Tabs

Add real app tabs:

- Warroom
- Brief
- Decisions
- Docs
- Forge

Tabs are local UI state. No routing is required for v0.3.

### Product Brief

Add a Brief tab for the active product.

Editable fields:

- one-liner
- audience
- problem
- promise
- pricing hypothesis
- primary milestone
- strategic constraint

Brief edits must persist to local storage.

### Decision Ledger Skeleton

Add a Decisions tab for the active product.

Required fields for a decision:

- title
- status: open, chosen, parked
- context
- options
- chosen path
- revisit trigger

v0.3 must allow adding a decision. Editing existing decisions can wait unless it falls out naturally.

### Docs Tab Placeholder

Add a Docs tab that names the launch knowledge assets Pendragon will track next.

It should be useful as a preview, not a full tracker yet.

### Forge Tab Placeholder

Add a Forge tab that summarizes what Forge will eventually build from the product room.

It should feel like a real module waiting for readiness, not a dead empty state.

### Storage Compatibility

Existing local workspaces must continue to load.

If an older product is missing `brief` or `decisions`, the app should merge in the seeded defaults without discarding local edits.

## Non-Goals

- Backend
- Auth
- AI generation
- Team comms integrations
- URL routing
- Import JSON
- Full docs asset tracker
- Editing existing decisions
- Forge site generation
- Multi-user collaboration

## UX Contract

### Warroom

The Warroom remains the default command view.

### Brief

Brief should feel like the product strategy card behind the launch.

It should be editable but not look like a settings page.

### Decisions

Decisions should make unresolved choices visible and reduce circular thinking.

The first version can be simple: a ledger plus one add form.

### Docs

Docs should show the launch knowledge surface Pendragon cares about:

- landing page
- beta instructions
- privacy/trust
- FAQ
- changelog
- support/contact

### Forge

Forge should preview the future output:

- landing page
- docs
- FAQ
- pricing
- changelog
- trust page

## Data Contract

Extend each product with:

```js
{
  brief: {
    problem: "...",
    promise: "...",
    pricingHypothesis: "...",
    primaryMilestone: "...",
    strategicConstraint: "..."
  },
  decisions: [
    {
      id: "decision-writing-quest-pricing",
      title: "Price early access at $19 or $29?",
      status: "open",
      context: "...",
      options: ["$19", "$29"],
      chosenPath: "",
      revisitTrigger: "..."
    }
  ]
}
```

Keep existing top-level product fields for now:

- `oneLiner`
- `user`
- `currentFocus`
- `decisionNeeded`
- `topRisk`

v0.3 should not split the model into separate stores yet.

## Acceptance Criteria

- The app still opens at `/app/`.
- Warroom remains the default tab.
- Brief, Decisions, Docs, and Forge tabs render real content.
- Brief edits persist after reload.
- Adding a decision persists after reload.
- Older local workspaces missing v0.3 fields still load.
- Mobile has no page-level horizontal overflow.
- Browser console has no errors in the verified flows.

## Future Module: Accountability Signals

Pendragon should eventually integrate with team communication tools such as Slack, Microsoft Teams, and Google Chat.

The job is not generic notifications. The job is launch-risk accountability.

Example:

> You have not made a decision on Task A. If a decision is not made by Friday, the release will move to at risk.

This belongs after Pendragon has durable decisions, owners, dates, release milestones, and risk rules.
