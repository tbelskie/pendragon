# Pendragon v0.4 Docs Tracker Plan

v0.4 makes the Docs tab operational.

The goal is not docs generation yet. The goal is to make launch knowledge visible enough that Writing Quest can move toward paid beta with fewer hidden trust gaps.

## Product Job

Pendragon v0.4 should answer:

> What launch knowledge assets are required, which ones are ready, and what is blocking trust?

## Why This Comes Next

Writing Quest's current launch risk is not branding.

It is trust:

- Can writers believe their work is saved?
- Can they export safely?
- Can they understand the beta?
- Can they install without fear?
- Can they contact support?

The docs tracker turns those worries into visible launch assets with status, priority, next move, proof link, and evidence.

## Required Scope

### Docs Asset Model

Each product gains `docsAssets`.

Required fields:

- `id`
- `title`
- `status`: `missing`, `drafting`, `blocked`, or `ready`
- `priority`: `Critical`, `High`, or `Medium`
- `purpose`
- `nextStep`
- `proofLink`
- `evidence`

### Docs Tab

Replace the placeholder Docs tab with:

- docs readiness percentage
- ready asset count
- critical open count
- blocked count
- drafting count
- editable asset cards
- save action that persists locally

### Readiness Sync

When the docs tracker is saved, sync the product's `Docs` readiness row to the tracker:

- `done` equals docs assets with `status: ready`
- `total` equals total docs assets

This makes Docs readiness less arbitrary over time.

### Storage Compatibility

Older local workspaces must gain seeded `docsAssets` without losing existing product, brief, decision, or room edits.

## Non-Goals

- AI generation
- Markdown export
- Docs site generation
- File writes into product repos
- Integrations
- Multi-user review
- Editing asset titles or priorities from the UI

## UX Contract

The Docs tab should feel like a launch-control surface, not a CMS.

Each asset should answer:

- What is this?
- Why does it matter?
- Is it missing, drafting, blocked, or ready?
- What is the next move?
- Where does the proof live?
- What evidence proves it exists?

## First Dogfood Pass

Writing Quest gets the most opinionated seed data:

- product positioning brief
- beta landing page
- save and export trust page
- local data promise
- beta install instructions
- writer FAQ
- support and feedback path

The tracker should keep the save/export blocker visible until the app behavior is trustworthy enough to document.

## Acceptance Criteria

- The app still opens at `/app/`.
- Older local workspaces load and gain seeded docs assets.
- The Docs tab renders real tracker data for all seeded products.
- Editing asset status, next move, proof link, and evidence persists after reload.
- Saving Docs updates the Docs readiness row.
- Forge treats Docs as ready only when at least one docs asset is ready.
- Mobile has no page-level horizontal overflow.
- Browser console has no errors in the verified flows.
