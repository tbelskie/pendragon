# Pendragon v0.1 Platform Scope

v0.1 is not a full SaaS product.

It is an internal-but-real app used to manage the launch of Writing Quest and the broader product portfolio.

## Goal

Make Pendragon real enough to dogfood immediately.

The app should reduce launch anxiety by answering:

> What matters this week, what decision is blocking us, and what risk could sink the launch?

## Required Scope

### Branded App Shell

- Lava Glass logo in header.
- Dark command-center UI.
- Portfolio navigation.
- Mobile can be usable, but desktop is primary.

### Setup Flow

- Create a product room.
- Select founder stage.
- Capture product one-liner, user, milestone, target date, repo URL, and biggest worry.
- Generate a stage-appropriate starting warroom.

### Portfolio

- Show product cards.
- Seed internal portfolio:
  - Writing Quest
  - Dot Primer
  - MockBizOps
  - Vinegar

### Product Warroom

- This week's focus.
- Next three actions.
- Readiness counts for Build, Docs, Launch, Trust, Revenue.
- Decision needed.
- Top risk.
- Current milestone.

### Decision Ledger

- Add/edit decisions.
- Capture context, options, chosen path, and revisit trigger.

### Docs Surface Plan

- Track launch/docs assets.
- Mark assets as needed, drafting, ready, or published.

### Local Persistence

- Browser local storage.
- Reset to demo data.
- Export JSON.
- Import JSON.

### Forge Placeholder

The Forge screen should exist, but not generate sites yet.

It should describe the future feature:

> Build a launch site from this product room.

## Explicit Non-Goals

- Auth.
- Backend.
- Billing.
- AI generation.
- GitHub integration.
- Real Forge site generation.
- Multi-user collaboration.
- Analytics integrations.
- Public onboarding polish.

## Dogfood Definition

v0.1 is successful if it can manage the next Writing Quest launch cycle better than Markdown alone.

Success means:

- current focus is obvious
- launch risks are visible
- decisions are captured
- docs gaps are trackable
- next actions are concrete
- the app is used weekly
