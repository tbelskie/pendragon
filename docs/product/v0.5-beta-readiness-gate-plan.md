# Pendragon v0.5 Beta Readiness Gate Plan

v0.5 adds the first real launch judgment surface.

The goal is to help Writing Quest answer a blunt operating question:

> Can this product credibly enter paid beta yet?

## Why This Comes Next

The Docs tracker made launch knowledge visible.

The next job is synthesis. Pendragon should not only track work. It should turn readiness, docs blockers, trust gaps, and unresolved decisions into a clear call.

## Product Job

The Gate tab should answer:

- Is the product ready for paid beta or launch?
- What is blocking the gate?
- What is merely at risk?
- What should be cleared first?
- Which room should the founder open next?

## Required Scope

### Gate Tab

Add a new app tab:

- Warroom
- Brief
- Decisions
- Docs
- Gate
- Forge

Gate sits before Forge because Forge should build from launch readiness, not vibes.

### Computed Verdict

Compute the gate from existing product room data:

- overall readiness
- Build readiness
- Trust readiness
- Launch readiness
- Revenue readiness
- docs readiness
- critical docs assets
- blocked docs assets
- unresolved decisions
- support or feedback path

No new backend and no AI judgment.

### Verdict States

The Gate can return:

- `blocked`: blockers exist
- `at-risk`: no blockers, but important launch risks remain
- `ready`: no blockers and no at-risk criteria

For Writing Quest, the expected initial state is blocked because save/export trust, beta landing page, and pricing are not ready.

### Criteria

Initial criteria:

- Critical launch docs
- Decision ledger
- Trust proof
- Build stability
- Launch surface
- Revenue path
- Support loop

Each criterion shows:

- status
- short evidence
- next move

### Next Moves

The Gate should produce a short ordered list from blockers first, then at-risk criteria.

This is the bridge from dashboard to action.

## Non-Goals

- Editable custom gate rules
- AI recommendations
- Calendar timelines
- Team approvals
- Release automation
- Forge generation
- External integrations

## UX Contract

The Gate tab should feel like an executive launch review for a solo founder.

It should be clear, calm, and somewhat unforgiving.

The founder should leave the screen knowing whether to:

- keep building
- fix trust
- decide pricing
- finish docs
- move forward

## Acceptance Criteria

- The app still opens at `/app/`.
- Gate tab renders for all products.
- Writing Quest initially shows a blocked paid beta gate.
- Gate uses existing readiness, docs, and decision data.
- Gate next moves link back to Docs, Decisions, and Warroom.
- Mobile has no page-level horizontal overflow.
- Browser console has no errors in the verified flows.
