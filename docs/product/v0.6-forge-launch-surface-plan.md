# v0.6 Forge Launch Surface Plan

Forge is the first Pendragon module that should feel like a real product wedge instead of a better product notebook.

The v0.6 goal is simple:

> Turn a product room into a credible first launch surface that a solo founder can inspect, refine, and export.

## Why This Matters

Most founder operating systems stop at organization. Forge should cross the line into production help.

The wedge is not "track launch tasks." The wedge is:

> I filled in my product room, and Pendragon gave me the first usable launch page.

That makes the rest of Pendragon feel necessary. The brief, docs tracker, decision ledger, and gate all become inputs to a generated surface instead of isolated database fields.

## v0.6 Scope

Ship a local-only Forge tab that generates:

- an in-app launch surface preview
- a standalone HTML draft export
- clear readiness signals for the inputs feeding the surface
- proof links from docs assets when safe external links exist
- missing-surface prompts based on open docs assets and unresolved decisions

## Inputs

Forge v0.6 uses only data Pendragon already tracks:

- product name
- stage and target date
- one-liner
- audience
- product brief
- decision ledger
- docs assets
- proof links
- beta readiness gate
- next actions

## Generated Surface Sections

The first generated surface should include:

- hero: name, one-liner, audience, primary milestone
- promise: problem, promise, strategic constraint
- beta status: gate verdict and gate score
- proof: ready docs assets and proof links
- launch gaps: open critical docs and unresolved decisions
- next actions: the current room actions

## Non-Goals

Do not add these in v0.6:

- account system
- deploy pipeline
- AI generation
- theme editor
- custom domains
- CMS editing
- persistent backend

Those are later product questions. v0.6 proves whether a founder would trust Pendragon to produce the first version of a launch surface from structured launch context.

## Dogfood Standard

Writing Quest is the proof object.

Forge v0.6 is useful only if it can help answer:

> What would the Writing Quest beta page say if we generated it from the current launch room today?

If the answer is embarrassing, that is good information. It means the room is missing launch truth, not that Forge failed.

## Success Criteria

- The Forge tab creates a readable launch-page preview for every seeded product.
- The Writing Quest preview exposes the save/export trust gap without burying it.
- The HTML export downloads as a standalone file.
- Proof links appear only when docs assets have safe `http` or `https` links.
- The module feels like an early product primitive, not a decorative placeholder.

## Later Shape

If v0.6 works, the next steps are:

- selectable launch surface types: waitlist, paid beta, docs hub, changelog
- import from repository metadata and product docs
- generated copy variants with founder approval
- GitHub Pages publishing
- custom docs site scaffolding from the product room
- one-click updates when the room changes
