# 0002: Stage-Aware Warroom

Date: 2026-05-28

## Decision

Pendragon should be stage-aware.

The founder's product stage determines the initial experience, readiness checklist, and primary job-to-be-done.

Stages:

- Idea
- Prototype
- Beta
- Launch
- Growth

## Context

Micro-SaaS founders arrive with different levels of product reality.

A founder with only an idea needs validation structure. A founder with a working beta needs trust, docs, pricing, and launch structure.

One generic dashboard would either overwhelm early users or underserve launch-ready users.

## First Wedge

The first ICP is:

> Solo founders with a working prototype trying to reach beta or first revenue.

This maps most strongly to Prototype, Beta, and Launch modes.

## Consequences

- Setup flow must ask stage questions.
- Demo data should include Writing Quest as a Beta Mode product.
- v0.1 should support all stages lightly, but optimize for Beta/Launch.

## Status

Accepted.
