# Clear First State

Clear First state should make Pendragon feel humane.

The queue is generated, but the founder still needs room to say:

> I handled this. Here is what I learned.

## Controls

Each open command item has:

- Founder note input
- Save note
- Mark resolved
- Open source module

Each resolved item has:

- Founder note input
- Save note
- Reopen
- Open source module

## Language

Use resolved or handled language carefully.

Resolved is a local queue state. It should not imply the product room is objectively fixed. The source module and product signals still own the deeper truth.

## Placement

The note editor lives inside each queue item.

This avoids a modal and keeps the founder in the command flow. It also makes the queue useful for dogfooding without adding a full task detail page.

## Recently Resolved

Resolved items move below the active queue into a compact details section.

This keeps the main queue focused while giving the founder a recovery path if they resolved the wrong item or want to keep editing the note.

## Visual Treatment

Resolved items should look quieter:

- lower opacity
- green rank number
- same card structure
- same routing affordance

The founder should understand the item is no longer active, but still part of the command record.
