# Clear First Command Queue

The Clear First queue is the pressure release valve for a solo founder.

It should make the founder feel:

> I know exactly what to unblock next.

## Surface

Clear First lives inside Portfolio, directly under the portfolio pulse metrics.

It is not a full task manager. It is a command queue generated from the product rooms.

## Item Anatomy

Each item shows:

- queue rank
- action label
- source module and product
- blocker title
- blocker detail
- product attention score
- direct route button

The first item is marked Start here. Urgent items are marked Unblock. Lower-severity items are still queued, but visually quieter.

## Source Modules

Queue item buttons should open:

- Setup
- Gate
- Docs
- Forge
- Evidence
- Decisions
- Warroom

The founder should never have to wonder where the work lives.

## Visual Direction

Clear First should be more operational than decorative:

- dense enough to scan
- clear ranks
- strong red-orange accent for urgency
- restrained blue for focus work
- subtle green for steady work
- compact action buttons

The queue sits between the command hero and product cards. The hero says what matters. The queue says what to do.

## Product Boundary

Do not turn this into generic task management yet.

The queue is generated from product-room truth. Manual tasks, assignments, reminders, and team notifications can exist later only if they preserve that contract.
