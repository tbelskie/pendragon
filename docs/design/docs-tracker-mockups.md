## Design Rules

- Keep cards to one column on desktop so each docs asset reads like a launch object, not a tile.
- Use small status chips instead of large colorful panels.
- Use orange for blocked launch risk, green for ready, blue for drafting, and muted gray for missing.
- Keep editable fields inside each asset card so the user does not jump into a separate settings page.
- Preserve the warroom feel: this is an operating surface, not a content management system.

## Product Meaning

Docs readiness is not a writing chore.

In Pendragon, docs readiness means:

> The founder can explain, prove, sell, support, and recover the product in public.

This is why the tracker belongs before Forge. Forge should generate from healthy launch knowledge, not from vibes.
