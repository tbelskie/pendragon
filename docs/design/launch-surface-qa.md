# Launch Surface QA

Launch Surface QA is the first judgment layer inside Forge.

It should feel like a sharp preflight check, not another dashboard.

## Layout

```text
Forge / Paid beta page

[Download HTML draft] [Edit Brief] [Open Docs]

Readiness signals
Surface strategy
Launch Surface Controls

Launch Surface QA
Not ready to share                         57%
Writing Quest still has 3 publishability gaps...

[Pass] Offer           [Fix] CTA
[Pass] Proof           [Pass] Trust claim
[Pass] Support path    [Fix] Critical docs
[Fix] Readiness gate

Clear First
CTA / Critical docs / Readiness gate

Generated launch surface
```

## Check Rules

- Offer passes when a saved or inferred offer exists.
- CTA passes when both CTA label and CTA URL exist.
- Proof passes when Forge has at least one attached evidence or docs proof link.
- Trust claim passes when the founder has saved an explicit claim.
- Support path passes when support docs are ready or a support URL exists.
- Critical docs passes when no critical docs assets are open.
- Readiness gate passes only when the product gate is ready.

## Tone

The copy should be direct but not punitive.

Use:

- "Not ready to share"
- "Share with caution"
- "Shareable draft"

Avoid:

- "Failed"
- "Invalid"
- "Broken"
- "Bad"

The founder should feel guided, not graded.

## Why This Exists

The readiness gate judges the product room.

Launch Surface QA judges the generated artifact.

That distinction matters because a product may be promising while its public surface is still dangerous to publish.
