# Create Product Flow

The create flow should feel like starting a product room, not filling out admin settings.

The founder should be able to create a room with very little certainty. Pendragon turns missing certainty into visible work after the room exists.

## Surface

Create mode lives inside Setup.

The page uses:

- Create Product Room hero
- setup signal score
- starter surface label
- New Product Intake form
- short explanation of what Pendragon creates

## Form Fields

The flow asks for:

- product name
- product type
- business model
- current goal
- stage mode
- target date
- status
- source link
- one-liner
- audience
- problem
- promise
- prototype/repo/users/charging signals
- pricing hypothesis
- traction or proof note
- current focus
- top risk

Only product name is required.

## Creation Behavior

On create:

- infer stage from the setup profile
- create a unique product id
- generate starter readiness rows
- generate starter docs assets
- create one source evidence link when supplied
- create one stage-defining decision
- route the founder into the new Warroom

## Starter Assets

Every new room gets:

- Positioning brief
- Source of truth
- Support and feedback path
- one stage-specific asset

Examples:

- Idea gets a validation sprint plan.
- Prototype gets an MVP scope note.
- Beta gets a beta landing page.
- Launch gets a paid launch page.
- Relaunch gets a relaunch page.
- Stalled gets a restart brief.
- Shipped gets a traction baseline.
- Growth gets a growth experiment note.

## Interaction Notes

- Cancel returns to the previously active product setup.
- Reset demo clears locally-created products.
- Export JSON includes created products.
- The New product nav item should remain visually distinct from real portfolio rooms.
