# Portfolio Command Center

The Portfolio Command Center is the first screen that treats Pendragon as the founder's operating system.

It should feel calm, premium, and decisive. A solo founder should be able to open it on Saturday morning and know where to put the next two hours.

## Surface Anatomy

The screen uses:

- Portfolio Command hero
- recommended focus product
- attention score
- portfolio pulse metrics
- ranked product cards
- direct routing actions

The design should avoid a dense dashboard wall. The job is prioritization, not observation.

## Hero

The hero answers:

> Where should I look first?

It shows the recommended product, the reason it rose to the top, and the next move. The attention score is visible but secondary to the recommendation.

## Metrics

The metrics are deliberately few:

- products
- average readiness
- blocked gates
- stage mismatches
- with proof

Each metric must be actionable. If the founder cannot use it to change behavior this week, it does not belong in v0.

## Product Cards

Each card shows:

- product name
- product one-liner
- attention score
- readiness percent
- gate score
- docs readiness
- proof count
- reason surfaced
- next move

The top card is labeled Recommended. Other cards show their stage.

## Interaction Rules

Card actions route directly to:

- Warroom for weekly execution
- Setup for stage and signal correction
- Forge for launch surface work

The Portfolio tab can change the active product without making the user manually pick it from the sidebar first.

## Visual Notes

The screen should stay in the Lava Glass world:

- black glass base
- red-orange command accents
- subtle green for healthy states
- restrained blue for focus states
- no decorative clutter

Cards and controls use the existing 8px radius system.
