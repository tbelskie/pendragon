# Launch Share Package

Launch Share Package is the first outbound artifact generated by Forge.

It should feel like the founder has a launch brief ready to carry into the world.

## Layout

```text
Forge / Paid beta page

Launch Surface QA
Not ready to share                         57%

Launch Share Package
Give the founder launch copy...            [Download Markdown]

[Primary launch post] [Copy]
...

[Short teaser] [Copy]
...

[Founder DM] [Copy]
...

Clear First                         Proof Links
[Fix] CTA                           [Repo] Writing Quest repo
[Fix] Critical docs
[Fix] Readiness gate

Next Moves
1. CTA: Add a CTA URL...
2. Critical docs: 2 critical docs assets...
```

## Copy Blocks

Primary launch post:

- audience
- one-liner
- offer
- promise
- CTA line
- QA status

Short teaser:

- product name
- one-liner
- current offer
- CTA or CTA warning

Founder DM:

- direct one-to-one ask
- target user
- offer
- CTA or missing CTA warning

## Rules

- Never hide QA blockers.
- Keep copy plain text.
- Do not invent proof.
- Do not invent a CTA link.
- Keep Markdown export complete enough to paste into a working doc.

## Why This Exists

The launch surface is for public inspection.

The share package is for founder action.

Both should come from the same source of truth so the founder is not reconciling a page, a checklist, and a launch doc by hand.
