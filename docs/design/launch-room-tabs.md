# Launch Room Tabs Design Direction

v0.3 adds depth without adding clutter.

The app should feel like one premium product room with multiple command surfaces, not five separate pages.

## Visual Rules

- Keep the Lava Glass shell.
- Use tabs as compact product controls, not giant navigation cards.
- Preserve the Warroom as the first screen.
- Keep forms contained inside premium panels.
- Do not introduce decorative backgrounds beyond restrained glass surfaces.
- Keep labels product-native and human: "Audience", "Promise", "Strategic constraint".

## Screen Direction

### Warroom

The current command view remains the launch heartbeat.

### Brief

The Brief tab should look like a founder strategy card:

- crisp input fields
- concise labels
- a saved-state row
- a strong read-mode summary

### Decisions

The Decisions tab should feel like a ledger, not a task board.

Use compact decision cards with:

- status chip
- title
- context
- options
- chosen path
- revisit trigger

### Docs

The Docs tab should preview the launch asset surface in a tight checklist-like layout.

### Forge

The Forge tab should make the future obvious:

> When the room is healthy enough, Forge turns it into launch assets.

## Interaction Rules

- Switching tabs must not reset unsaved edits by surprise.
- Save actions must be explicit.
- Local-only state must remain visible.
- Empty states should feel intentional, not broken.
