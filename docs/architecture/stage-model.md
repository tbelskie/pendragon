# Founder Stage Model

Pendragon should be useful across founder stages, but the first wedge is founders with working prototypes moving toward beta or first revenue.

The setup flow should identify the founder's current stage and generate an appropriate product room.

## Stage 1: Idea Mode

For founders with a concept but no working product.

Primary job:

> Turn a vague idea into a product brief and validation sprint.

Pendragon should produce:

- product brief
- audience hypothesis
- problem statement
- promise
- differentiator
- riskiest assumptions
- validation plan
- kill criteria

Commercial note: idea-stage users may be lower willingness-to-pay. This mode should exist, but it is not the initial wedge.

## Stage 2: Prototype Mode

For founders building or vibe-coding an MVP.

Primary job:

> Prevent shapeless building.

Pendragon should produce:

- smallest lovable v1
- scope cuts
- build readiness checklist
- docs checklist
- decision ledger
- launch blockers
- "not yet" list

## Stage 3: Beta Mode

For founders with something usable, but not confidently launchable.

Primary job:

> Make the product trustworthy enough for real users.

Pendragon should produce:

- beta readiness checklist
- onboarding path
- support path
- privacy/trust docs
- pricing hypothesis
- feedback loop
- top trust risk
- next user plan

This is the strongest first product mode.

## Stage 4: Launch Mode

For founders close to charging, announcing, or relaunching.

Primary job:

> Turn the product into a sellable launch.

Pendragon should produce:

- launch plan
- landing page/docs plan
- pricing decision
- announcement assets
- GTM experiment
- launch checklist
- metrics plan
- retrospective date

## Stage 5: Growth Mode

For founders with users or revenue.

Primary job:

> Turn launch signal into a repeatable learning loop.

Pendragon should produce:

- retention questions
- conversion review
- customer feedback synthesis
- docs/support gaps
- next experiment
- pricing revisit
- product iteration focus

## Setup Questions

The first setup flow should ask:

1. What are you building?
2. What stage is it in?
3. Do you have a working prototype?
4. Do you have a repo?
5. Do real users have access?
6. Are you charging?
7. What is the next milestone?
8. What are you most worried about?

The answers determine the initial room.
