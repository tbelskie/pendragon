# Docs-As-Code Product Site

Use this to build a product website and docs surface that can evolve with the product.

## Minimum Surface

- home page
- getting started
- FAQ
- changelog
- privacy/trust
- support/contact

## Developer Product Add-Ons

- API reference
- quickstart
- examples
- playground
- SDK notes
- status page

## Operating Rule

Docs must be treated as product surface, not launch garnish.
