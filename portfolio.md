# Portfolio Map

The portfolio gives Pendragon real material to work against. Each product teaches a different launch pattern.

## Writing Quest

Retro writing desktop app for indie writers and hobbyists.

Primary lessons:

- paid desktop launch
- seasonal launch timing
- emotional positioning
- UX writing for creative tools
- local-first trust
- license and export readiness

## Dot Primer

Chrome extension for focused deep work sessions.

Primary lessons:

- browser extension distribution
- Chrome Web Store listing optimization
- retention measurement
- lightweight analytics
- focus/productivity positioning
- small-product maintenance

## Vinegar

Impulse-shopping blocker for intentional spending.

Primary lessons:

- behavior-change product design
- privacy/trust messaging
- consumer onboarding
- friction that feels helpful, not punitive
- possible mobile path

## MockBizOps

Realistic fake business APIs for serious prototypes, demos, tutorials, and docs.

Primary lessons:

- developer product launch
- API documentation
- docs-as-code workflows
- public demo infrastructure
- usage/pricing limits
- technical content and examples

## Portfolio Rule

Only one product gets active build focus at a time.

Other products can receive:

- bug fixes
- analytics checks
- documentation cleanup
- one-day validation tasks
- tiny marketing/listing experiments

If a new idea appears, it enters the parking lot until the current active bet reaches a defined decision point.
